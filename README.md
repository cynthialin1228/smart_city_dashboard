## Setup — Conda Environment

### 1. Install Conda

If you don't have Conda, install [Miniconda](https://docs.conda.io/en/latest/miniconda.html):

```bash
# macOS (Apple Silicon or Intel)
brew install --cask miniconda
# or download installer from https://docs.conda.io/en/latest/miniconda.html
```

### 2. Create the environment

```bash
conda create -n smart_city python=3.11 -y
conda activate smart_city
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

> On first run, YOLOv8 will automatically download the model weights (~22 MB for `yolov8s.pt`). An internet connection is required.

## Running the Pipeline

### Full pipeline (detect → analyze → serve dashboard)

```bash
conda activate smart_city
bash run_pipeline.sh path/to/your_video.mp4
```

Then open **http://localhost:8080** in your browser.

### Options

```bash
# Use a larger, more accurate model (slower)
bash run_pipeline.sh video_20min.mp4 yolov8m.pt

# Default (fast, good accuracy)
bash run_pipeline.sh video_20min.mp4 yolov8s.pt
```

### Run stages individually

```bash
# Stage 1+2: Detection & Tracking
python pipeline/detect_and_track.py \
    --video  video_20min.mp4 \
    --model  yolov8s.pt \
    --skip   2 \
    --conf   0.35 \
    --output pipeline/tracking_raw.json

# Stage 3: Analysis & Recommendations
python pipeline/analyze_traffic.py \
    --input  pipeline/tracking_raw.json \
    --output dashboard/data/traffic_data.json

# Stage 4: Serve dashboard
cd dashboard && python -m http.server 8080
```

### Preview dashboard without running the pipeline

A sample `dashboard/data/traffic_data.json` is included. Just serve the dashboard directly:

```bash
cd dashboard && python -m http.server 8080
```

Then open **http://localhost:8080**.

## Pipeline Arguments

### `detect_and_track.py`

| Argument   | Default              | Description                                      |
|------------|----------------------|--------------------------------------------------|
| `--video`  | *(required)*         | Path to input video file                         |
| `--model`  | `yolov8s.pt`         | YOLOv8 variant: `yolov8n/s/m/l/x.pt`            |
| `--skip`   | `2`                  | Frames to skip between detections (speed tuning) |
| `--conf`   | `0.35`               | Detection confidence threshold (0.0–1.0)         |
| `--output` | `pipeline/tracking_raw.json` | Output JSON path                        |

### `analyze_traffic.py`

| Argument   | Default                            | Description              |
|------------|------------------------------------|--------------------------|
| `--input`  | `pipeline/tracking_raw.json`       | Raw tracking JSON        |
| `--output` | `dashboard/data/traffic_data.json` | Dashboard output JSON    |

## Dashboard Features

| Panel | Description |
|-------|-------------|
| **KPI Cards** | Total vehicles, congestion index, peak time, duration, traffic level |
| **Vehicle Distribution** | Donut chart — Cars/SUVs, Motorcycles, Trucks, Buses, Pedestrians |
| **Traffic Timeline** | Stacked bar chart per minute |
| **Directional Flow** | Per counting line vehicle breakdown |
| **Congestion Heatmap** | Color-coded cell per minute (hover for details) |
| **Recommendations** | Rule-based actionable advice (signal timing, lane policy, etc.) |
| **Line Details Table** | Per-line counts, congestion index, and level badge |

**Language toggle**: Click the `中文 / English` button in the top-right corner.

## Troubleshooting

**`FileNotFoundError: Cannot open video`**  
→ Check the video path. Run from the `smart_city_dashboard/` directory.

**Dashboard shows "Dashboard data not found"**  
→ The pipeline hasn't been run yet, or `dashboard/data/traffic_data.json` is missing. Use the included sample or run the pipeline first.

**YOLOv8 model download fails**  
→ Ensure internet access on first run. Model is cached to `~/.config/Ultralytics/` after download.

**Slow processing**  
→ Increase `--skip` (e.g., `--skip 4`) or use `yolov8n.pt` (nano, fastest).
